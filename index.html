💾 Where your data lives

Everything in this app — employees, shifts, projects, leaves — is stored locally in this browser on this computer. Nothing is uploaded to any cloud service. Attendance records specifically are stored in IndexedDB (a larger, more robust local database built into the browser) so the app can comfortably handle very large imports — tens or even hundreds of thousands of records — without hitting the small ~5–10MB ceiling that plain browser storage has.

Because it's local-only, this data will not follow you to a different browser or computer, and can be lost if browser data is cleared. Download a backup regularly, especially after a large import.

📦 Share as One File (data included)

Sending the plain .html file over WhatsApp/email only sends the empty app — none of your data goes with it, because data lives in this browser only. Use this instead: it saves a new .html file with all of your current data baked right in. Send that file — whoever opens it (any PC, any browser) will automatically see the same employees, attendance, shifts and projects, no separate backup file, no restore step needed.

⬇ Download a Backup

Saves everything — employees, attendance, shifts, projects, departments, leaves — into a single .json file you keep safe (e.g. Google Drive, a USB drive, email to yourself).

⬆ Restore from a Backup

Warning: this replaces all current data in the app with whatever is in the backup file. Consider downloading a fresh backup first if you have unsaved changes.

🧹 Clear Attendance for a Date Range

Use this to wipe out incorrect/dummy attendance for specific dates (e.g. records saved before real biometric data was imported) so you can re-import clean data. This only removes attendance records — employees, shifts and projects are untouched.

🧾 Attendance-Only Backup & Import

Use this when you only want to move attendance records — not employees, shifts, or projects. Unlike the full restore above, importing here merges into whatever is already in the app (matching existing employees by Employee Code, updating same-day records, adding new ones) instead of replacing everything. Employees or projects mentioned in the file that don't already exist here will be auto-created so no record is lost.

🏗️ Project Backup & Restore

Each project can also be backed up and restored on its own — useful for archiving a completed project or moving one project's data to another install of this app. Go to Projects, click ⬇ Backup on any project card to download just that project (its employees, attendance, shifts and leaves), and use ⬆ Restore Project Backup at the top of the Projects page to bring one back in — it merges in without touching your other projects.

👥 Manage Team Roles

Create the person's login first (Supabase → Authentication → Users), then assign their role here. Admin — full access, including managing roles. Manager — can mark attendance and manage employees/leave/shifts/projects, but can't manage roles. Viewer — read-only access everywhere. Optionally tick specific modules below to restrict a Manager/Viewer to only those — leave all unchecked for full access. Admins always see every module.

Import Attendance from Device Files — By Project

✅ Each row below is tagged to a Project — click the file field on a row and select multiple files at once (hold Ctrl/Cmd or Shift while choosing) to upload several ZKTeco machine exports into that same project in one go. You can also reopen the file picker to add more files to the same row — they'll be added to the list below it, and each can be removed individually with ✕.
✅ Files usually carry their own date per punch, so the Date field can be left blank. Only set it if your file has no date at all, or you want every record in that file forced onto one specific day (e.g. a single day's export).
✅ Need to import only a specific period from a file (e.g. one payroll cycle)? Use the Date Range fields to keep only punches that fall between those two dates — everything outside the range is dropped automatically.
✅ Need files split across different sites/branches? Click "+ Add Another Project File" to add more rows, each with its own Project and its own batch of files.
✅ Employees are automatically matched by Biometric ID (ZK Pin) or Employee Code — no manual selection needed.
✅ Original punch times from each file are preserved exactly as recorded on the device.
✅ Employees on rotating or unassigned shifts? Turn on Auto-Detect shift per punch below so Present/Late is judged against whichever shift's start time is closest to each check-in.
When checked, each punch's Present/Late status is decided by whichever defined shift's start time is closest to that check-in — instead of the employee's fixed/rostered shift. Useful for employees who rotate shifts or don't have one assigned.
